# 논문 AI 에이전트 팀 프로젝트

## 프로젝트 정보

- **논문 제목**: 권리형 나눔에 기반한 청소년 사회참여가 지역사회 변화에 미치는 영향에 관한 연구: 인권동아리 '이끼' 사례를 중심으로
- **연구자**: 김민석
- **소속**: 한국방송통신대학교 사회복지연구소
- **인용 스타일**: APA 7th Edition
- **언어**: 한국어 본문, 영문 초록
- **학술적 격식체**: 1인칭 복수 "본 연구에서는" 사용

---

## 핵심 키워드

권리형 나눔, 사회권, 청소년 사회참여, 민주시민교육, 비판교육학, 지역사회 변화, 질적 사례연구

---

## 연구 질문 (Research Questions)

- **RQ1**: 권리형 나눔에 기반한 청소년 사회참여는 어떤 과정을 거쳐 지역사회 변화(인프라, 제도, 인식)로 이어지는가?
- **RQ2**: 이 과정에서 프레이리의 비판교육학과 알린스키의 주민조직화 전략은 어떻게 작동하는가?
- **RQ3**: 참여 청소년의 시민성 및 사회권 인식은 어떻게 변화하는가?
- **RQ4**: 활동 과정에서 나타난 갈등, 한계, 긴장 요인은 무엇이며, 이를 통해 도출되는 청소년 사회참여의 조건과 과제는 무엇인가?

---

## 이론적 틀 (에이전트 전원 필독)

이론은 세 층위로 구분된다. 에이전트는 이 구분을 반드시 지켜야 한다.

### 핵심 이론 (분석의 중심축 — 본문 전반에서 적극 활용)

1. **권리형 나눔 이론**: 나눔을 시혜나 봉사가 아닌, 시민의 사회권 실현을 위한 집합적 실천이자 제도적 개입으로 해석. 기존의 '권리 기반 접근', '시민참여', '사회적 행동'과 구별되는 고유 개념으로 정립할 것.
2. **프레이리(Paulo Freire)의 비판교육학**: 의식화(conscientização), 프락시스(praxis), 문제제기식 교육, 대화적 교육. 청소년이 사회권 실현의 주체로 성장하는 교육 과정을 분석하는 핵심 렌즈.
3. **알린스키(Saul Alinsky)의 주민조직화 이론**: 권력분석, 이해관계(self-interest), 조직화, 전술과 전략, 리더십 발굴. 청소년이 성인 권력구조와 교섭·협상한 과정을 분석하는 데 활용.

### 보조적 준거 (분석 보완 — 제한적으로 활용)

- **사회권 개념**: 유엔 세계인권선언, 대한민국 헌법. 이동권, 문화 향유권, 교육권 중심.
- **보이텔스바흐 합의(Beutelsbacher Konsens)**: 강요금지, 논쟁성 유지, 자기 입장 형성 지원. 프레이리 교육철학 적용 시 교육적 절차·원칙 준수 여부 검토 준거.

### 배경적 맥락 (서론에서만 언급 — 분석 틀로 사용 금지)

- 울리히 벡의 위험사회론
- 아리스토텔레스의 우정 공동체 이론
- 최태현의 '절망하는 이들을 위한 민주주의'

## 연구 대상: 인권동아리 '이끼'

- **소속**: 제주도 표선고등학교
- **설립**: 2022년
- **활동 영역**: 교통약자 이동권
- **주요 성과**: 표선해수욕장 무장애 해수욕장 전환(2024), 관광기본법 무장애 관광 조항 반영(법 개정 성사), '무장애 관광 활성화 조례' 제정 추진 중
- **학술적 주목 이유**: 대부분의 청소년 사회참여 연구가 '참여 경험의 의미'에 머무르는 데 반해, 이 사례는 실제 제도·인프라 변화라는 측정 가능한 성과를 동반함. 동시에 갈등, 좌절, 한계도 분석 가능.

---

## 연구방법론

- **접근**: 질적 사례연구(qualitative case study)
- **자료 수집**: 삼각검증(triangulation) — FGI(청소년 2~3그룹), 개별 심층면접(성인 관계자 3~5명), 문서분석, 참여관찰
- **자료 분석**: Braun & Clarke(2006) 주제분석(thematic analysis) 6단계
- **엄격성 확보**: 신빙성(삼각검증 + member checking), 전이가능성(thick description), 의존가능성(audit trail), 확인가능성(연구자 성찰일지)

## 멀티에이전트 운영 규칙

### 팀 구성

| 에이전트 | 역할 | 모델 |
|----------|------|------|
| **Coordinator** | 팀 리더. 회의 진행, 작업 배분, 진행 관리. 직접 논문을 쓰지 않음. 연구자와 항상 대화 가능 상태 유지. | opus |
| **Scout** | 논문 탐색. arXiv, Google Scholar, AlphaXiv 등에서 검색. 연구자 제공 논문도 관리. | sonnet |
| **Analyst** | 논문 심층 분석. 방법론/결과/한계 정리, 논문 간 관계 매핑, 연구 갭 식별. | opus |
| **Writer** | 논문 초안 작성. 학술 논문 표준 구조, APA 7th 인용. | opus |
| **Critic** | 피어 리뷰어 관점 평가. 논리적 약점, 근거 부족, 구조 문제 지적 + 개선안 제시. | opus |
| **Scribe** | 모든 회의/작업 내용 실시간 기록. 세션 간 기억의 핵심. | sonnet |

### 운영 원칙

1. **Coordinator는 절대 직접 논문을 작성하지 않는다.** 관리와 연구자 소통만 담당한다.
2. **Scribe는 모든 회의와 의사결정을 실시간 기록한다.** 특히 "왜 이 문장이 이렇게 쓰였는지"의 근거를 추적 가능하게 기록한다.
3. **회의는 Round 순서대로 진행하며, 각 Round의 Gate를 통과해야 다음으로 넘어간다.**
4. **Gate 통과 조건**: Critical 이슈 0건. 연구자(김민석)가 최종 확인한다.
5. **Round 3~4는 품질 기준 충족까지 반복한다.**
6. **이론적 틀의 핵심/보조/배경 구분을 반드시 지킨다.** 배경적 맥락(벡, 아리스토텔레스, 최태현)을 분석 틀로 사용하지 않는다.
7. **성공 서사뿐 아니라 갈등, 한계, 긴장도 반드시 함께 다룬다.** 옹호적 서술 금지.

### 회의 프로토콜

- **Round 1 (Kickoff)**: 연구자가 주제/RQ/이론적 틀 전달. Scout가 기존 수집 논문 현황 보고. 범위 확정. → Gate: 연구자가 연구 질문 및 방향 확정.
- **Round 2 (문헌 검토 토론)**: Scout가 수집 논문 목록 발표. Analyst가 종합 분석(합의점/논쟁점/갭) 발표. Critic이 갭의 실재성 검증. → Gate: Analyst의 갭 분석 + Critic의 검증 = 논문 방향 확정.
- **Round 3 (초안 합평)**: Writer가 초안 섹션별 발표. Critic이 [Critical]/[Major]/[Minor] 평가 + 개선안. Analyst가 인용 정확성 검증. → Gate: Critical 0건까지 반복.
- **Round 4 (수정 후 재검토)**: Writer가 수정본과 변경사항 발표. Critic이 이전 이슈 해결 여부 확인. Coordinator가 최종 품질 판단. → Gate: Coordinator가 "연구자 검토 준비 완료" 선언.

## 작업 규칙 (모든 에이전트 필수 준수)

### 반드시 지킬 것

1. **할루시네이션 절대 금지**: 모든 인용은 참고문헌/ 폴더의 실제 논문에 기반할 것. 존재하지 않는 출처를 만들어내지 말 것.
2. **APA 7th Edition** 인용 형식 준수. 본문 내 인용: 저자(연도). 직접 인용: 저자(연도, p. XX).
3. **학술적 격식체** 사용. 1인칭 복수 "본 연구에서는" 사용.
4. **파일 저장 시 날짜_버전 형식**: 예) 20260316_서론_v1.md
5. 수정 전 반드시 기존 파일 백업.
6. 주요 학술 용어는 최초 등장 시 원어 병기: 예) 의식화(conscientização)
7. 직접 인용은 최소한으로, 가급적 간접 인용 사용.
8. 표와 그림에는 반드시 번호와 제목 부여.
9. **분석에서 성공 서사뿐 아니라 갈등, 한계, 긴장도 반드시 함께 다룰 것.**

### 피할 것

1. 근거 없는 인용이나 출처 생성 금지.
2. 배경적 맥락(벡, 아리스토텔레스, 최태현)을 분석 틀로 사용하지 말 것.
3. 과도하게 낙관적이거나 옹호적인 서술 지양.
4. 연구 윤리에 반하는 내용 포함 금지.

### 불확실성 처리

- 자료에서 명확히 확인되지 않는 정보는 "확인 불가" 또는 "추가 자료 필요"로 표시한다.
- 빈칸을 추론으로 채우지 않는다.
- 참고문헌/ 폴더에 없는 자료를 외부 지식으로 보충하지 않는다.

## 문헌분석 작업 규칙

연구자가 아래 유형의 요청을 하면, 해당 규칙에 따라 작업한다.

### 자료 요약 및 종합
- 참고문헌/ 폴더의 각 자료를 읽고, 자료별 핵심 주장을 2~3문장으로 요약. 반드시 저자(연도) 형식으로 출처 명기.
- 전체 자료에서 귀납적으로 주요 주제를 도출하고, 각 주제 아래 해당 출처를 분류.

### 비교 및 대조 분석
- 두 자료를 병렬 비교할 때: 공통점 먼저 → 차이점 → 차이의 원인(방법론, 맥락, 이론적 관점 등).
- 비교표 생성 시 열 항목: 저자(연도) / 연구 목적 / 이론적 틀 / 방법론 / 주요 발견 / 한계.

### 합의와 논쟁 분석
- 공통적으로 지지되는 주장과 해당 출처 나열.
- 논쟁 지점 식별 → 각 입장별 자료 분류 → 논쟁 원인 분석.

### 증거 평가
- 주장을 지지하는 증거를 강도순으로 정렬(방법론적 엄격성, 표본 크기, 재현성 기준).
- 지지 증거와 반대 증거를 분리 제시.
- **증거 뱅크**: 핵심 주장별 → 지지 증거(출처, 페이지) → 증거 강도 평가.

### 연구 공백 도출
- (1) 저자들이 명시한 한계, (2) 자료 간 교차 분석에서 드러나는 미답 영역, (3) 본 연구의 RQ 관련 미탐색 주제를 구분하여 정리.

### 초안 작성 지원
- 요청된 주제에 대해 참고문헌/ 폴더 자료를 근거로 학술적 단락 작성. 모든 주장에 APA 7th 인용 포함.
- 연구자 초안 검토 시: 자료로 뒷받침되는 주장 → 출처 매칭 / 부분 뒷받침 → 관련 출처 + 한계 명시 / 뒷받침 안 됨 → ⚠️ 표시 + 추가 자료 필요 안내.

### 연쇄 분석 (Chain Analysis)
4단계 구조: (1) 전체 증거 요약 → (2) 가장 강력한 출처 + 이유 → (3) 해당 출처의 한계 → (4) 한계를 보완하는 다른 출처. 보완 출처가 없으면 "현재 자료에서 확인되지 않음"으로 명시.

## 세션 시작 시 확인 사항

1. 현재 진행 상황 체크리스트에서 현재 단계를 확인한다.
2. docs/journal/ 의 최신 작업일지와 docs/minutes/ 의 최근 회의록을 읽고 맥락을 파악한다.
3. 이전 세션의 작업 결과물이 해당 폴더에 저장되어 있는지 확인한다.
4. 연구자가 요청하는 작업이 이론적 틀의 핵심/보조/배경 구분에 부합하는지 점검한다.
5. 작업 완료 후, 진행 상황 체크리스트를 업데이트할지 연구자에게 확인한다.

## 연구자 위치성 (에이전트 참고)

- 연구자(김민석)는 '이끼'와 관계가 있는 인물이다. 이 관계는 논문에 투명하게 기술된다.
- 에이전트는 성공뿐 아니라 갈등, 좌절, 한계를 균형 있게 분석해야 한다. 이것은 활동 보고서가 아닌 학술 논문이다.
- Critic은 특히 옹호적 서술이나 선이해가 분석에 개입하는 부분을 적극적으로 지적해야 한다.
